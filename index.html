Cloud sync (Supabase)

Optional. Save your Statistics to a Supabase project so they're available across devices.

How to set up
  1. Create a free project at supabase.com.
  2. Open the project's SQL Editor and run the schema below.
  3. Open Project Settings → API, copy the Project URL and the anon public key.
  4. Paste them into the form below, set a User ID (any string), and click "Test & sync".
create table if not exists focus_sessions (
  id bigserial primary key,
  user_id text not null default 'me',
  date text not null,
  timestamp bigint not null,
  ext_score int,
  int_score int,
  tasks_total int,
  tasks_done int,
  completed_tasks jsonb,
  pct int,
  created_at timestamptz default now()
);
alter table focus_sessions enable row level security;
create policy "anon read"   on focus_sessions for select using (true);
create policy "anon insert" on focus_sessions for insert with check (true);
create policy "anon delete" on focus_sessions for delete using (true);

Note: with the anon key + permissive policies above, anyone holding the key can read/write rows. For a personal app this is fine; use Supabase Auth for stricter isolation.
